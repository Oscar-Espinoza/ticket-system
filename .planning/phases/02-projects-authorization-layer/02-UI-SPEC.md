# Phase 2 — UI Design Contract

> Visual and interaction contract for the project list, create-project dialog,
> and project detail shell. Generated by gsd-ui-researcher. Verified by
> gsd-ui-checker.

---

## Scope

Three frontend surfaces are built in Phase 2. The authorization primitive
(`requireProjectMember`) is server-only with no UI of its own.

1. **Dashboard project list** — renders inside the `{children}` seam of
   `src/app/dashboard/page.tsx`. Clickable project cards + empty state with CTA.
2. **Create-project Dialog** — shadcn `Dialog` triggered by "New project" button.
   Two fields: project name and ticket key.
3. **Project detail page** — `/dashboard/projects/[id]`: header with name + key
   badge, and an empty ticket-list placeholder. Non-members receive `notFound()`
   (no access-denied UI to design — D-15).

## Spacing Scale

Inherited from Phase 1. No new exceptions for Phase 2.

| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Card internal padding (compact rows), button icon gap |

## Typography

Inherited from Phase 1. Sizes and weights are additive — the Phase 2 Display
role (28px) enters use here for the first time (reserved in Phase 1).

All sizes use Geist Sans. Declared weights: **400** (normal) and **600**
(semibold). No other weights.

| Role | Size | Weight | Line Height | Usage in Phase 2 |
|------|------|--------|-------------|-------------------|
| Body | 14px | 400 | 1.5 | Card ticket counts, dialog helper text, empty state body |
| Label | 14px | 600 | 1.4 | Dialog field labels, card role badge text |
| Heading | 20px | 600 | 1.3 | "Projects" section heading above the card grid; project detail page header |
| Display | 28px | 600 | 1.2 | Not used in Phase 2 (reserved for Phase 3+) |

Note: Project name on a card uses the Heading size (20px/600) only if the card
layout has room for prominence. If cards are compact, use Label (14px/600) for
the name. The executor must choose based on actual card height — see Card Layout
below.

Inherited from Phase 1 OKLCH tokens (shadcn radix-nova preset).

| Role | Tailwind Token | Usage in Phase 2 |
|------|----------------|-------------------|
| Dominant (60%) | `background` | Dashboard page background, dialog backdrop, detail page background |

**Accent (`primary`) reserved for:**
- "New project" primary CTA button (dashboard empty state and top of list)

Accent is NOT used for: project cards, badges, ticket count numbers, the
cancel button in the dialog, the project detail header.

**Role badges on cards:**
- Owner: `Badge` variant `secondary` — text "Owner"
- Member: `Badge` variant `outline` — text "Member"

**Ticket key badge (on cards and detail header):**
- `Badge` variant `secondary` — monospace-styled text (use `font-mono`)

## Page / Surface Layouts

### Dashboard — Project List (renders into `{children}` seam)

```
┌──── main content (container mx-auto max-w-4xl px-6 py-8) ────┐
│                                                                 │
│  [Greeting — from Phase 1 shell — h2 20px/600]                │
│  [GitHub status badge — from Phase 1 shell]                    │
│                                                                 │
│  ── Phase 2 content begins here (children slot) ──            │
│                                                                 │
│  ┌── section header row ──────────────────────────────────┐   │
│  │ Projects  [h3 16px/600]        [New project  Button]   │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌── project card ──────────────────────────────────────┐     │
│  │ [Project name 14px/600]     [KEY badge] [Role badge] │     │
│  │ [N open · N resolved  14px/400 muted-foreground]     │     │
│  └──────────────────────────────────────────────────────┘     │
│  ┌── project card ──────────────────────────────────────┐     │
│  │ ...                                                   │     │
│  └──────────────────────────────────────────────────────┘     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

- Section header: `flex items-center justify-between mt-8 mb-4` — 32px top
  margin separates from the greeting block above.
- Section label: `h3` tag, 16px/600 (between Label and Heading roles).
- "New project" button: shadcn `Button` variant `default` (accent fill).
  Contains `Plus` icon (lucide-react) prepended to text.
- Cards: `flex flex-col gap-3` vertical stack. Each card is a full-width
  shadcn `Card` wrapping a `CardContent` with `p-4` (md spacing).
- Card layout: single row `flex items-center justify-between gap-4`.
  - Left: project name (14px/600) + ticket-key badge + role badge in a
    `flex items-center gap-2` group.
  - Right: ticket counts in a `flex items-center gap-3` group.
- Ticket counts: `{N} open · {N} resolved` as `span` elements,
  `text-sm text-muted-foreground`.
- The entire card is a Next.js `<Link>` to `/dashboard/projects/{id}` — the
  card cursor is `cursor-pointer` and the card has `hover:bg-accent/50`
  transition (shadcn built-in via `Card` with `asChild` or wrapping anchor).
- Card ordering: newest project first (ORDER BY `createdAt` DESC).

```
┌──── empty state panel ──────────────────────────────────────┐
│                                                               │
│           No projects yet                                     │

│                                                               │
│                [New project  Button]                          │
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

- Panel: `flex flex-col items-center justify-center py-12 gap-4` (48px
  vertical padding = 2xl spacing token).
- Heading: "No projects yet" — 16px/600, `text-foreground`.
- Body: "Create your first project to get started." — 14px/400,
  `text-muted-foreground`.
- CTA button: identical to the header row "New project" button — same variant,
  same icon — opens the same Dialog.
- The header row "New project" button is ALWAYS visible (even in empty state),
  so the empty-state CTA is a second affordance, not the only one.

### Create-Project Dialog

Triggered by either "New project" button. Single-step inline form.

```
┌──── Dialog (max-w-md) ──────────────────────────────────────┐
│  Create project                         [X close]            │
│  ─────────────────────────────────────────────────────────── │
│                                                               │
│  Project name                                                 │
│  [Input  placeholder="My project"]                           │
│                                                               │
│  Ticket key                                                   │
│  [Input  placeholder="APP"  max 6 chars]                     │
│  2–6 uppercase letters, unique across all projects.          │
│  [inline error if duplicate or invalid]                       │
│                                                               │
│  ─────────────────────────────────────────────────────────── │
│                          [Cancel]  [Create project]          │
└───────────────────────────────────────────────────────────────┘
```

- Dialog component: shadcn `Dialog` with `DialogContent`, `DialogHeader`,
  `DialogTitle`, `DialogDescription`, `DialogFooter`.
- Title: "Create project" — shadcn `DialogTitle` (inherits 20px/600 heading).
- Description: not rendered visually (set for screen readers only via
  `DialogDescription` with `className="sr-only"`).
- Max width: `sm:max-w-md` (448px).
- Fields: each wrapped in `div` with `flex flex-col gap-1.5` (6px gap between
  label and input — shadcn default, acceptable minor exception to 4px scale).
- Field spacing between the two fields: `mt-4` (16px = md token).
- Ticket key input behavior:
  - Auto-uppercase on every keystroke (`onChange` transforms value to
    `.toUpperCase().replace(/[^A-Z]/g, '')`, capped at 6 chars).
  - Helper text below input: "2–6 uppercase letters, unique across all projects."
    — 14px/400, `text-muted-foreground`, always visible.
  - Inline error (duplicate key or length violation) appears below helper text
    in `text-destructive text-sm`.
- Project name input: no special transform. Placeholder: "My project".
- Footer: `flex justify-end gap-2`.
  - Cancel: `Button` variant `outline`.
  - Submit: `Button` variant `default` (accent). Text: "Create project".
  - Submit button shows `Loader2` icon with `animate-spin` during submission,
    disabled during loading. Text changes to "Creating…".
- On success: dialog closes, project list revalidates (server revalidation),
  user sees the new card at the top of the list.
- On failure (non-duplicate): inline error below relevant field using
  `text-destructive text-sm`.

## Interaction Contracts

### Ticket Key Input Transform

- On every `onChange` event: transform value to `.toUpperCase()`, strip all
  non-A-Z characters, cap at 6 characters.
- This happens client-side on every keystroke — the user never sees lowercase
  or non-letter characters.
- Validation (2-char minimum, uniqueness) runs on submit, not on blur.

### Dialog Open / Close

- Dialog opens on click of either "New project" button.
- Dialog closes on: (a) successful submit with server revalidation, (b) Cancel
  button click, (c) clicking the overlay, (d) pressing Escape.
- Form state resets on close (controlled state or uncontrolled with `key` prop
  reset).

### Form Validation Timing (matches Phase 1 pattern)

- Validate on **submit** only — not on blur.
- Show all field errors simultaneously after a failed submit.
- Clear a field's error as soon as the user starts typing in that field.

### Loading States

- Submit button: `Loader2` icon (`animate-spin`) replaces the leading icon (if
  any), button is `disabled`, text changes to "Creating…".
- No page-level spinner. No skeleton.

### Post-Create Behavior

- On success: dialog closes, dashboard project list updates (server
  revalidation via `revalidatePath('/dashboard')`), new project card appears at
  the top of the list.
- No success toast. The card appearing is the confirmation.

### Project Card Interaction

- Entire card is a link — no separate "Open" button.
- Hover state: `hover:bg-accent/50` (or equivalent shadcn card hover).
- Focus state: standard shadcn focus ring (keyboard navigable).
- No secondary actions on the card (no kebab menu, no inline delete — Phase 4+).

### Back Navigation (Project Detail)

- "← Back to projects" is a standard Next.js `<Link>` — no JavaScript history
  manipulation. Always navigates to `/dashboard`.

## Accessibility Requirements

All Phase 1 requirements carry forward. Phase 2 additions:

- Dialog has `DialogTitle` for screen readers (required by Radix `Dialog`).
- Ticket key helper text is associated with the input via `aria-describedby`.
- Inline errors are associated with their input via `aria-describedby`
  (separate from helper text — concatenate IDs or use separate elements).
- Card links have meaningful accessible names: "Project name (KEY) — N open,
  N resolved" via `aria-label` or visible text alone is sufficient if the card
  text provides full context.
- Back link has visible icon + text (not icon-only).
