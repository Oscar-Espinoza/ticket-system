    (owner unremovable, D-33).

    (ACKNOWLEDGED — checker WARNING 3: the role-check SELECT is INTENTIONALLY kept
    separate from the DELETE rather than collapsed into a single conditional delete
    `db.delete(...).where(and(eq(projectId), eq(userId), eq(role,'member')))` with a
    `rowsAffected === 0` check. Rationale: (1) the explicit role read produces the
    precise "The project owner cannot be removed." error copy the UI-SPEC requires —
    a count-based rejection cannot distinguish an owner row from an already-gone row;
    (2) it avoids relying on neon-http `rowsAffected`/result-shape semantics. This
    matches 03-PATTERNS.md "owner guard placement; single db.delete". Leave as-is.)

    `db.delete(projectMembers).where(and(eq(projectMembers.projectId, projectId), eq(projectMembers.userId, memberUserId)))`
    (single delete — neon-http, no transaction). Then
    `revalidatePath(\`/dashboard/projects/${projectId}/members\`)` and return
    `{ success: true }`. Import `and`, `eq` from 'drizzle-orm'.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-12 | Elevation of Privilege | removeMember | mitigate | requireProjectOwner before any delete; ProjectAccessError → { error: 'Forbidden' }. UI hides controls for non-owners but the server is the boundary (CVE-2025-29927). |
| T-03-13 | Denial of Service | owner removes self/owner | mitigate | Reject self-remove (memberUserId === session.user.id) and reject deleting an 'owner' row — project can never become ownerless/locked-out (D-33). |
| T-03-14 | Tampering | IDOR remove across projects | mitigate | requireProjectOwner is scoped to (projectId, session.user.id); delete WHERE filters both projectId AND memberUserId — no cross-tenant removal. |
| T-03-15 | Information Disclosure | removed member retains access | mitigate | Per-request authorization via requireProjectMember (membership never in JWT) → next request after delete throws → notFound()/403 (D-34). |
| T-03-SC | Tampering | npx shadcn add alert-dialog | mitigate | Official shadcn registry only; blocking-human checkpoint precedes the install; alert-dialog is a standard documented primitive. |
</threat_model>
