  <files>src/app/actions/join.ts</files>
  <behavior>
    - With a valid unexpired token + new user: inserts one project_member row role 'member', returns success
    - Re-running for an already-member (or owner) user: inserts NO new row (count stays 1), returns success
    - Concurrent double-join hitting the unique constraint (23505): treated as already a member (success), not an error
    - Expired or unknown token: returns an error state, inserts nothing
    - No session: returns not-authenticated error
  </behavior>
  <read_first>
    - src/app/actions/projects.ts (the whole file — session resolution, crypto.randomUUID, 23505 detection + re-throw, error-state shape)
    - src/lib/project-access.ts (ProjectAccessError; requireProjectMember if used for the skip-check)
    - src/db/schema.ts (invitations.expiresAt NOT NULL; projectMembers unique(project_id,user_id) from Plan 01)
    - .planning/phases/03-membership-invite-links/03-CONTEXT.md (D-27 explicit POST, D-28 invalid token, D-29 idempotent join + 23505 backstop)
    - .planning/phases/03-membership-invite-links/03-PATTERNS.md (23505 mapping → "already a member" → redirect)
  </read_first>
  <action>
    Create `'use server'` file exporting type
    `JoinProjectState = { error?: string; success?: boolean; projectId?: string }`
    and `export async function joinProject(prevState, formData): Promise<JoinProjectState>`.
    Read `token` from formData. Resolve session; if no user return
    `{ error: 'Not authenticated' }`. Look up the invitation:
    `db.select().from(invitations).where(and(eq(invitations.token, token), gt(invitations.expiresAt, new Date()))).limit(1)`
    (import `and`, `eq`, `gt` from 'drizzle-orm'). If no row → return
    `{ error: 'invalid' }` (do NOT leak project details, D-28). Otherwise derive
    `projectId`. Check-then-insert idempotency: query project_member for
    (projectId, session.user.id); if a row exists (member OR owner) skip the insert
    and return `{ success: true, projectId }`. Else
    `db.insert(projectMembers).values({ id: crypto.randomUUID(), projectId, userId: session.user.id, role: 'member', createdAt: new Date() })`
    inside try/catch: map SQLSTATE 23505 (concurrent double-join) to
    `{ success: true, projectId }` (treat as already a member — the race-safe
    backstop, D-29); re-throw any other code. On success
    `revalidatePath(\`/dashboard/projects/${projectId}\`)` and return
    `{ success: true, projectId }`. Do NOT call `redirect()` inside the action — let
    the client button perform navigation from the returned projectId so the action
    stays testable. (Re-throwing non-23505 errors is REQUIRED.)
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public internet → /invite/[token] | first un-guarded route; token + session both untrusted |
| link prefetchers / unfurlers → invite URL | bots may GET the URL; must not auto-join |
| client → joinProject action | token from FormData untrusted; concurrent double-submit possible |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-08 | Tampering | auto-join on visit | mitigate | Join is an explicit POST Server Action triggered only by the submit button (D-27); GET on /invite/[token] only renders the landing — no mutation on a safe method. |
| T-03-09 | Information Disclosure | invalid token page | mitigate | State C returns 200 with a generic message and no project internals; never notFound() (avoids confirming token existence) per D-28. |
| T-03-10 | Tampering | duplicate membership under concurrent join | mitigate | check-then-insert + 23505 backstop on the live unique(project_id,user_id) constraint → second insert resolves to "already a member" success (D-29). |
| T-03-11 | Elevation of Privilege | expired link reuse | mitigate | Invitation lookup filters `expiresAt > now`; expired tokens yield State C / { error: 'invalid' } and never insert (D-24). |
| T-03-SC | Tampering | package installs | accept | No package-manager installs in this plan. |
</threat_model>
