# Phase 2 ŌĆö UI Design Contract

> Visual and interaction contract for the project list, create-project dialog,
> and project detail shell. Generated by gsd-ui-researcher. Verified by
> gsd-ui-checker.

---

## Scope

Three frontend surfaces are built in Phase 2. The authorization primitive
(`requireProjectMember`) is server-only with no UI of its own.

1. **Dashboard project list** ŌĆö renders inside the `{children}` seam of
   `src/app/dashboard/page.tsx`. Clickable project cards + empty state with CTA.
2. **Create-project Dialog** ŌĆö shadcn `Dialog` triggered by "New project" button.
   Two fields: project name and ticket key.
3. **Project detail page** ŌĆö `/dashboard/projects/[id]`: header with name + key
   badge, and an empty ticket-list placeholder. Non-members receive `notFound()`
   (no access-denied UI to design ŌĆö D-15).

## Typography

Inherited from Phase 1. Sizes and weights are additive ŌĆö the Phase 2 Display
role (28px) enters use here for the first time (reserved in Phase 1). The
Subheading role (16px/600) is introduced in Phase 2 for section labels and
empty-state headings that need more visual weight than Body but less than Heading.

All sizes use Geist Sans. Declared weights: **400** (normal) and **600**
(semibold). No other weights.

| Role | Size | Weight | Line Height | Usage in Phase 2 |
|------|------|--------|-------------|-------------------|
| Body | 14px | 400 | 1.5 | Card ticket counts, dialog helper text, empty state body, back link |
| Label | 14px | 600 | 1.4 | Dialog field labels, card role badge text, card project name |
| Subheading | 16px | 600 | 1.4 | "Projects" section heading, empty-state heading "No projects yet", empty-ticket-placeholder heading "No tickets yet" |
| Heading | 20px | 600 | 1.3 | Project detail page header (project name h1), dialog title |
| Display | 28px | 600 | 1.2 | Not used in Phase 2 (reserved for Phase 3+) |

Note: Project name on a card uses the Label size (14px/600) so the card remains
compact. The section heading and empty-state headings use Subheading (16px/600)
to provide clear separation from 14px/400 body text without reaching Heading
prominence. Total distinct sizes in use: 14, 16, 20, 28.

**Ticket key badge (on cards and detail header):**

## Interaction Contracts

### Ticket Key Input Transform

- On every `onChange` event: transform value to `.toUpperCase()`, strip all
  non-A-Z characters, cap at 6 characters.
- This happens client-side on every keystroke ŌĆö the user never sees lowercase
  or non-letter characters.
- Validation (2-char minimum, uniqueness) runs on submit, not on blur.

### Dialog Open / Close

- Dialog opens on click of either "New project" button.
- Dialog closes on: (a) successful submit with server revalidation, (b) Discard
  button click, (c) clicking the overlay, (d) pressing Escape.
- Form state resets on close (controlled state or uncontrolled with `key` prop
  reset).

### Form Validation Timing (matches Phase 1 pattern)

- Validate on **submit** only ŌĆö not on blur.
- Show all field errors simultaneously after a failed submit.
- Clear a field's error as soon as the user starts typing in that field.

### Loading States

- Submit button: `Loader2` icon (`animate-spin`) replaces the leading icon (if
  any), button is `disabled`, text changes to "CreatingŌĆ”".
- No page-level spinner. No skeleton.

### Post-Create Behavior

- On success: dialog closes, dashboard project list updates (server
  revalidation via `revalidatePath('/dashboard')`), new project card appears at
  the top of the list.
- No success toast. The card appearing is the confirmation.

### Project Card Interaction

- Entire card is a link ŌĆö no separate "Open" button.
- Hover state: `hover:bg-accent/50` (or equivalent shadcn card hover).
- Focus state: standard shadcn focus ring (keyboard navigable).
- No secondary actions on the card (no kebab menu, no inline delete ŌĆö Phase 4+).

### Back Navigation (Project Detail)

- "ŌåÉ Back to projects" is a standard Next.js `<Link>` ŌĆö no JavaScript history
  manipulation. Always navigates to `/dashboard`.
