## Spacing Scale

Multiples of 4 only. Matches Tailwind v4 default scale as used across Phase 2 pages.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps (`gap-1`), inline badge padding |
| sm | 8px | Compact list item padding, button icon gap |
| md | 16px | Default element spacing, card content padding |
| lg | 24px | Section breaks within a page (mb-6) |
| xl | 32px | Between major sections (mb-8) |
| 2xl | 48px | Vertical centering padding (py-12) |
| 3xl | 64px | Empty-state tall card (py-16) |

Exceptions:
- Header height: 56px (h-14, 3.5rem) — inherited from Phase 2 layout, not on scale but established pattern.
- Touch targets for icon-only buttons (Copy, Regenerate): minimum 44px height via `h-9` + padding on Button component.

## Typography

Matches Phase 2 established patterns exactly. Do not introduce new sizes.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / helper text | 14px (text-sm) | 400 (normal) | 1.5 |
| Label / card metadata | 14px (text-sm) | 600 (semibold, font-semibold) | 1.5 |
| Section heading | 16px (text-base) | 600 (semibold, font-semibold) | 1.2 |
| Page heading (h1) | 20px (text-xl) | 600 (semibold, font-semibold) | 1.2 |

Weights: 400 (regular) and 600 (semibold) only.

Monospace: `font-mono` (Geist Mono) used for ticket-key badges. Also use `font-mono` for the rendered invite link URL in the invite panel, so the token is visually distinct.

Source: Phase 2 code — `text-xl font-semibold` for h1, `text-base font-semibold` for section heads, `text-sm` for body/labels, `text-sm text-muted-foreground` for secondary text.

## Color

All values expressed as CSS custom properties (OKLCH tokens from globals.css). Do not hardcode hex values.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-background` | Page background, full-bleed container |
| Secondary (30%) | `bg-card` / `bg-muted` | Member roster cards, invite panel card, sidebar |
| Accent (10%) | `bg-primary` / `bg-primary-foreground` | Primary button fill only |
| Destructive | `text-destructive` / destructive Button variant | Remove member button and confirmation dialog |

Accent (`bg-primary`) is reserved for:
- The "Join project" primary CTA button on the `/invite/[token]` landing page.
- The "Generate invite link" button (when no link exists yet, owner view only).

All other interactive elements (Copy, Regenerate, Back link) use `variant="outline"` or `variant="ghost"` — never accent fill.

Role badges:
- Owner: `<Badge variant="secondary">Owner</Badge>` — matches Phase 2 project-list pattern.
- Member: `<Badge variant="outline">Member</Badge>` — matches Phase 2 project-list pattern.

Source: `src/app/globals.css` (OKLCH tokens), `src/components/project-list.tsx` (badge pattern).

## Page Inventory

Two new routes for this phase:

### `/dashboard/projects/[id]/members`

Full members management page. Guarded by `requireProjectMember` → `notFound()`.

Layout sections (top to bottom):
1. **Shared header** — identical to Phase 2's `h-14` header (`"Ticket System"` + user email).
2. **Main container** — `container mx-auto max-w-4xl px-6 py-8` (matches project detail page).
3. **Back link** — `← Back to project` linking to `/dashboard/projects/[id]`. Same style as Phase 2: `text-sm text-muted-foreground hover:text-foreground flex items-center gap-1 mb-6`.
4. **Page heading** — `<h1 className="text-xl font-semibold mb-8">Members</h1>`.
5. **Invite panel** (owner-only, conditionally rendered) — Card containing:
   - Label: "Invite link"
   - Read-only Input showing the full invite URL (`font-mono text-sm`)
   - `Copy` button (outline variant, icon: `Copy` from lucide)
   - `Regenerate` button (outline variant, icon: `RefreshCw` from lucide)
   - If no link exists: "Generate invite link" primary button instead of the URL field.
   - `<Separator className="my-6" />` below invite panel before the roster.
6. **Roster section heading** — `<h2 className="text-base font-semibold mb-4">Team members</h2>`.
7. **Member list** — `flex flex-col gap-3`. Each row is a `Card > CardContent p-4`:
   - Left: member name (`text-sm font-semibold`) + role Badge.
   - Right (owner-only): `Remove` button (`variant="ghost"` with `text-destructive` color, icon: `UserMinus` from lucide).

### `/invite/[token]`

Public page (outside `/dashboard` tree). No shared layout or auth guard on the route itself — the page handles auth state internally.

Three render states:

**State A — Valid token, logged-in user, not yet a member:**
- Centered card layout: `min-h-screen flex items-center justify-center bg-background`.
- Card width: `w-full max-w-md`.
- Heading: `<h1 className="text-xl font-semibold">Join {projectName}</h1>`.
- Body: `<p className="text-sm text-muted-foreground mt-2">You've been invited to join this project.</p>`.
- CTA: `<Button className="w-full mt-6">Join project</Button>` (primary variant).

**State B — Valid token, logged-out visitor:**
- Same centered card layout.
- Heading: `<h1 className="text-xl font-semibold">You've been invited</h1>`.
- Body: `<p className="text-sm text-muted-foreground mt-2">Sign in to join {projectName}.</p>`.
- CTA: `<Button className="w-full mt-6" asChild><Link href="/login?redirect=/invite/[token]">Sign in to continue</Link></Button>` (primary variant).

**State C — Invalid / expired token:**
- Same centered card layout.
- Heading: `<h1 className="text-xl font-semibold">Invalid invite link</h1>`.
- Body: `<p className="text-sm text-muted-foreground mt-2">This invite link is invalid or has expired. Ask the project owner for a new link.</p>`.
- No CTA button. Link: `<Link href="/dashboard" className="text-sm text-muted-foreground underline">Go to dashboard</Link>` (only if logged in).

**State D — Already a member (idempotent visit):**
- Server Action redirects immediately to `/dashboard/projects/[id]`. No special page state needed — the redirect is silent from the user's perspective.

## Interaction Contracts

### Copy invite link

Trigger: "Copy" button click.
Mechanism: `navigator.clipboard.writeText(inviteUrl)`.
Feedback: Button label transitions from "Copy" to "Copied!" for 2000ms, then reverts. Use `useState` with `setTimeout`. No toast needed — inline feedback is sufficient for a single-action panel.
Failure: If `navigator.clipboard` is unavailable (HTTP context), silently fallback to selecting the Input content (`inputRef.current?.select()`).

### Regenerate invite link

Trigger: "Regenerate" button click.
Mechanism: Owner-only Server Action. No confirmation step — regeneration is reversible (generates a new link, the old one simply stops working; no data is lost).
Feedback: `revalidatePath` causes the page to re-render with the new URL in the Input field. No optimistic UI needed.
Loading state: Button shows spinner icon (`Loader2 className="animate-spin"`) while the Server Action is pending. Use React `useTransition` or `useFormStatus`.

### Generate invite link (first time)

Trigger: "Generate invite link" primary button click (visible when no link exists).
Mechanism: Same Server Action as regenerate, just the first call.
Feedback: Page re-renders with invite panel showing the new URL.

### Join project

Trigger: "Join project" primary button click on `/invite/[token]`.
Mechanism: Server Action POST. Idempotent — if already a member, redirects to project without error.
Feedback: On success, hard redirect to `/dashboard/projects/[id]`. On error (expired link after page load), show inline error message below the button: `<p className="text-sm text-destructive mt-2">Unable to join. The invite link may have expired.</p>`.

### Remove member

Trigger: "Remove" button click (owner-only, per-member row).
Confirmation: AlertDialog with:
  - Title: "Remove member?"
  - Description: "**{memberName}** will immediately lose access to this project. This cannot be undone."
  - Cancel button: `variant="outline"`, label "Cancel".
  - Confirm button: `variant="destructive"`, label "Remove member".
Mechanism: Owner-only Server Action on AlertDialog confirm. `revalidatePath` the members page.
Feedback: Page re-renders without the removed member row. No toast needed.
