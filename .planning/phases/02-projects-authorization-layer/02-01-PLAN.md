| Boundary | Description |
|----------|-------------|
| authenticated client → Server Component / Server Action | A logged-in user supplies a `projectId` they may not belong to (untrusted input). |
| app code → neon-http `db` | All project-scoped reads cross here; must be gated by membership first. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | `requireProjectMember` / any project-scoped read | mitigate | Membership SELECT runs FIRST; throws `ProjectAccessError` before any project-scoped SELECT (403-before-DB, MEM-06, success criterion #4). Verified by the MEM-06 tests in Task 2/3. |
| T-02-02 | Elevation of Privilege | callers passing arbitrary `projectId` | mitigate | `requireProjectMember(projectId, userId)` is the single authorization seam every downstream action/page must call before touching project data (pitfall #3 convention). |
| T-02-SC | Tampering | `npx shadcn@latest add dialog` | accept | shadcn CLI copies source from the official registry (no npm install); same CLI vetted in Phase 1. No `[ASSUMED]`/`[SUS]` package introduced. |
