## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> /api/auth/[...all] | Untrusted credentials/email enter the auth handler |
| browser -> /dashboard | Untrusted request attempts to reach a protected resource |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Elevation of privilege | Protected route guard | mitigate | Server-side `auth.api.getSession()` in dashboard/layout.tsx + redirect; NEVER middleware-only (CVE-2025-29927) |
| T-01-05 | Spoofing | Session management | mitigate | Better Auth issues a fresh session token on login (default) — guards against session fixation; database sessions, httpOnly cookie |
| T-01-06 | Information disclosure | Password storage | mitigate | Better Auth default password hashing (scrypt); bcryptjs path stays on Node runtime — no `runtime='edge'` in auth handlers (pitfall 4) |
| T-01-07 | Information disclosure | Duplicate-email error | accept | Phase 1 uses no email verification, so 422 + USER_ALREADY_EXISTS is acceptable enumeration surface (RESEARCH Security Domain); revisit if email verification added |
| T-01-08 | Tampering | Input validation | mitigate | Better Auth validates email format + minPasswordLength 8; client validates on submit for UX |
</threat_model>
