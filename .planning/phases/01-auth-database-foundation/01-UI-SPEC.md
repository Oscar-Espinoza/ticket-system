# Phase 1 — UI Design Contract

> Visual and interaction contract for auth pages and the minimal dashboard.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Scope

Phase 1 UI surface is intentionally minimal — three screens only:

1. `/login` — email/password form + "Continue with GitHub" OAuth button
2. `/signup` — email/password registration form + "Continue with GitHub" OAuth button
3. `/dashboard` — authenticated landing: greeting + user email/name + GitHub-connected status badge + logout button

The `/dashboard` layout is a seam for Phase 2 project-list; no project-list UI is built here. (Source: D-09, CONTEXT.md)

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, checkbox gaps |
| sm | 8px | Input internal padding, button icon gap |
| md | 16px | Form field vertical rhythm, card padding |
| lg | 24px | Auth card internal padding, section breaks |
| xl | 32px | Card max-width gutter, vertical centering offset |
| 2xl | 48px | Auth page vertical padding (top/bottom of centered card) |
| 3xl | 64px | Full-page vertical centering on desktop |

Exceptions:

- Touch targets for the logout button and OAuth button: minimum 44px height (WCAG 2.5.8 target size)

## Typography

All sizes use Geist Sans. No custom font loading needed — Next.js scaffold includes Geist via `next/font`.

Declared weights: **400** (normal) and **600** (semibold). No other weights are used in Phase 1.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Form helper text, status descriptions |
| Label | 14px | 600 | 1.4 | Form labels, button text |
| Heading | 20px | 600 | 1.3 | Page headings (e.g. "Sign in", "Create account") |
| Display | 28px | 600 when used | 1.2 | Not used in Phase 1 — reserved for Phase 2+ |

Note: shadcn new-york style ships sensible defaults that match these values for `Input`, `Button`, and `Label` — the executor must not override them.

## Color

Using shadcn new-york defaults. All colors in OKLCH (Tailwind v4 CSS-first format).

| Role | Tailwind Token | Usage |
|------|----------------|-------|
| Dominant (60%) | `background` | Page background, auth page background |
| Secondary (30%) | `card` / `muted` | Auth card surface, input backgrounds |
| Accent (10%) | `primary` | Primary CTA buttons only (see reserved list below) |
| Destructive | `destructive` | Logout confirmation (if added), inline error text |
| Border | `border` | Input borders, card borders |
| Muted foreground | `muted-foreground` | Placeholder text, helper text, divider labels |

**Accent (`primary`) reserved for:**
- "Create account" submit button
- "Sign in" submit button
- "Continue with GitHub" OAuth button (using shadcn `Button` variant `default`, with GitHub icon prepended via lucide-react `Github`)

Accent is NOT used for: links, labels, status badges, the logout button.

**Logout button:** Uses shadcn `Button` variant `ghost` — no accent color. This visually de-emphasizes logout relative to primary actions.

**GitHub-connected status badge:**
- Connected: shadcn `Badge` variant `secondary` with text "GitHub connected" and `CheckCircle` icon from lucide-react
- Not connected: shadcn `Badge` variant `outline` with text "GitHub not connected" and `CircleOff` icon from lucide-react

## Page Layouts

### `/login` and `/signup` — Centered Auth Card

Primary focal point: the full-width primary CTA button at the bottom of the auth card.

```
┌─────────────────── viewport (100vh) ────────────────────┐
│                                                           │
│           ┌─── auth card (max-w-sm, ~384px) ───┐        │
│           │  [App name / logo]                   │        │
│           │  [Page heading — 20px/600]            │        │
│           │  [Subheading — 14px/400]              │        │
│           │                                       │        │
│           │  [Continue with GitHub Button]        │        │
│           │                                       │        │
│           │  ─── or ───                           │        │
│           │                                       │        │
│           │  Email [Input]                        │        │
│           │  Password [Input type=password]       │        │
│           │  [inline error if present]            │        │
│           │                                       │        │
│           │  [Primary CTA Button — full width]    │        │
│           │                                       │        │
│           │  [Switch link: "Don't have an account │        │
│           │   Sign up" / "Have an account? Sign in"]│       │
│           └───────────────────────────────────────┘        │
│                                                           │
└───────────────────────────────────────────────────────────┘
```

- Card uses `Card`, `CardHeader`, `CardContent` from shadcn
- Card centered with flexbox: `flex min-h-screen items-center justify-center`
- Card width: `w-full max-w-sm` with `px-6` horizontal padding on mobile
- GitHub OAuth button appears ABOVE the email/password form (reduces friction for OAuth-first flow)
- Separator with "or" label sits between GitHub button and email/password fields

### Error States (inline, below relevant input)

| Trigger | Copy |
|---------|------|
| Duplicate email on signup | An account with this email already exists. Sign in instead. |
| Invalid credentials on login | Invalid email or password. |
| Password too short (< 8 chars) | Password must be at least 8 characters. |

### Destructive Actions

| Action | Approach |
|--------|----------|
| Logout | No confirmation dialog — single click on ghost "Log out" button. Session is immediately cleared and user is redirected to `/login`. Logout is low-stakes enough that a confirmation would add friction without protecting anything. |

## Interaction Contracts

### Form Validation Timing

- Validate on **submit** only in Phase 1 (not on blur) — reduces noise for first-time users
- Display all field errors simultaneously after a failed submit
- Clear a field's error as soon as the user starts typing in that field

### Loading States

- Primary CTA buttons show a `Loader2` icon (lucide-react) with `animate-spin` during form submission
- Button is `disabled` during loading to prevent double-submit
- Text changes: "Sign in" → "Signing in…" / "Create account" → "Creating account…"

### OAuth Redirect

- "Continue with GitHub" button triggers Better Auth's GitHub sign-in; user is redirected to GitHub consent and then back to `/dashboard` on success
- No loading spinner needed — the browser navigates away immediately

### Post-Auth Redirects

| Event | Redirect |
|-------|----------|
| Successful login | `/dashboard` |
| Successful signup | `/dashboard` |
| Successful GitHub OAuth | `/dashboard` |
| Logout | `/login` |
| Unauthenticated access to protected route | `/login` (server-side redirect via auth guard, not middleware — D-10, CONTEXT.md) |
| Authenticated user visiting `/login` or `/signup` | `/dashboard` (prevent visiting auth pages while signed in) |

## Accessibility Requirements

- All form inputs have associated `<label>` elements (shadcn `Label` component) — not just placeholder text
- Inline error messages are associated with their input via `aria-describedby`
- Buttons have visible focus rings (shadcn default)
- Color is never the sole conveyor of information (error state uses text + color)
- Minimum contrast ratio 4.5:1 for body text (shadcn new-york palette satisfies this)
